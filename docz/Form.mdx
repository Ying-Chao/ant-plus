# Form 表单

## 使用

```js
import { Form } from 'antx';
```

## 示例

<Playground>
  {() => {
    /**
     * Ant PLus Form Demo
     */
    class Demo extends Component {
      constructor() {
        this.onSubmit = this.onSubmit.bind(this);
      }
      onSubmit()  {
        const { form } = this.props;
        form.validateFields((err, values) => {
          if (!err) {
            console.log(values);
          }
        });
      };

      render() {
        const { form } = this.props;
        return (
          <Form api={form} data={{ name: 'Emily', gender: 2 }} onSubmit={this.onSubmit}>
            <Input
              label="用户名"
              id="name"
              rules={['required', 'string', 'max=10']}
              max={10}
              msg="full"
            />
            <Select
              label="性别"
              id="gender"
              rules={['required', 'number']}
              data={[{ id: 1, type: '男' }, { id: 2, type: '女' }]}
              keys={['id', 'type']}
              msg="short"
              search
            />
            <Button htmlType="submit">提交</Button>
          </Form>
        );
      }
    }
    const WrappedDemo = Form.create()(Demo);
    return <WrappedDemo />;

}}

</Playground>

## API

### 1. Form

其它 Props 与 [Ant Design Form](https://ant.design/components/form-cn/) 完全一致，以下为增强 Props。

<PropsTable of={Form} />

### 2. 表单控件 Props

Ant Plus 最核心的使用方式 —— 直接在表单控件的 Props 中，传入相关 props。如下：

```jsx harmony
<Input
  label="用户名"
  id="username"
  colon
  after={<span>请输入正确的信息</span>}
  rules={['required', 'string']}
  initialValue="Emily"
  normalize={(value) => value}
/>
```

以下为表单控件可用的 Props：

** ① Form.Item 相关可用属性**

| 属性           | 说明                                              |
| -------------- | ------------------------------------------------- |
| colon          | Form `colon` 属性之外，自定义某个域的冒号是否显示 |
| extra          | 设置检验（不建议使用）                            |
| hasFeedback    | 设置检验（不建议使用）                            |
| help           | 设置检验（不建议使用）                            |
| label          | 名称                                              |
| labelCol       | 布局（不建议使用）                                |
| required       | 设置检验（不建议使用）                            |
| validateStatus | 设置检验（不建议使用）                            |
| wrapperCol     | 布局（不建议使用）                                |

_不建议使用 `extra`、`hasFeedback`、`help`、`required`、`validateStatus`，建议使用 `rules` 来设置校验信息。 不建议使用 `labelCol`、`wrapperCol`，建议使用 CSS 来布局，可减少大量纯布局用的 `<div>`。_

** ② form.getFieldDecorator(id, options) 相关可用属性**

| 属性              | 说明                                                           |
| ----------------- | -------------------------------------------------------------- |
| id                | `id`（表单域唯一标识，用于区分表单控件与普通组件，请勿被占用） |
| getValueFromEvent | `options.getValueFromEvent`                                    |
| initialValue      | `options.initialValue`                                         |
| normalize         | `options.normalize`                                            |
| preserve          | `options.preserve`                                             |
| rules             | `options.rules`（见下文 **3. rules** 使用方式）                |
| trigger           | `options.trigger`                                              |
| validateFirst     | `options.validateFirst`                                        |
| validateTrigger   | `options.validateTrigger`                                      |
| valuePropName     | `options.valuePropName`                                        |
| hidden            | `options.hidden`（来自 `rc-form`）                             |
| getValueProps     | `options.getValueProps`（来自 `rc-form`）                      |
| validate          | `options.validate`（来自 `rc-form`）                           |

** ③ 其它可用属性**

| 属性      | 说明                                                                                                               |
| --------- | ------------------------------------------------------------------------------------------------------------------ |
| className | `label` 存在时，`className` 将传给 `<Form.Item>`。此外，Ant Plus 会添加 `id` 的值到 `class` 列表中，方便自定义扩展 |
| style     | `label` 存在时，`style` 将传给 `<Form.Item>`                                                                       |  |
| before    | 表单域前缀信息（`string | ReactNode`），`label` 存在时有效，建议使用 CSS 来布局                                    |
| after     | 表单域后缀信息（`string | ReactNode`），`label` 存在时有效，建议使用 CSS 来布局                                    |
| hide      | 是否隐藏表单域（页面上不可见，但仍可收集到值）                                                                     |

### 3. rules

Ant Plus 提供了一种简洁的校验规则生成方式，传统 Ant Design Form 中，需要在 `rules` 数组中传入对象，如下：

```jsx harmony
form.getFieldDecorator('username', {
  rules: [
    { required: true, message: '用户名不得为空' },
    { type: 'string', whitespace: true, message: '用户名格式有误' },
    { max: 10, message: '不得超过 10 个字' },
  ],
})(<Input />);
```

而使用 Ant Plus，只需要传入「短语」字符串即可，如下：

```jsx harmony
<Input id="username" rules={['required', 'string', 'max=10']} />
```

Ant Plus 提供了以下「短语」形式的校验规则，配合下文 **4. Form.setConfig(config)**，可自定义校验提示。

| 规则     | 使用方式   | 校验         | 默认提示           |
| -------- | ---------- | ------------ | ------------------ |
| required | 'required' | 必填         | 'XXX 不得为空'     |
| string   | 'string'   | 字符串       | 'XXX 格式有误'     |
| number   | 'number'   | 数值         | 'XXX 格式有误'     |
| array    | 'array'    | 数组         | 'XXX 格式有误'     |
| email    | 'email'    | 邮箱         | 'XXX 格式有误'     |
| max      | 'max=10'   | 最大字符长度 | '不得超过 10 个字' |
| min      | 'min=2'    | 最小字符长度 | '不得少于 2 个字'  |
| phone    | 'phone'    | 手机号       | 'XXX 格式有误'     |
| id       | 'id'       | 身份证       | 'XXX 格式有误'     |

同时，遵循「渐进增强」的理念，当遇到未在以上「短语」中列出的规则，或需要自定义规则时，仍可使用传统方式，即传入对象，字符串与对象两种形式可搭配使用。

_注意：`max` 与 `min` 的使用方式为传入例如 `"max=10"`、`"min=2"`，以 `"="` 分割。上文校验提示中的 `"XXX"` 代表表单域的 `label`。_

### 4. Form.setConfig(config)

用于设置 Input 与 Select 的 `placeholder` 与校验提示，需在项目入口文件中调用。

`config` 对象可选配置项：

| 参数     | 默认提示              | 说明                                                                                                                                    |
| -------- | --------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| input    | '请输入'              | Input 的 `placeholder`，`msg` 为 `"short"` `"full"` 时有效（`"short"` => `"请输入"`，`"full"` 将在后面添加 `label`，如 "请输入用户名"） |
| select   | '请选择'              | Select 的 `placeholder`，`msg` 为 `"short"` `"full"` 时有效（`"short"` => `"请选择"`，`"full"` 将在后面添加 `label`，如 "请选择性别"）  |
| required | '{label}不得为空'     | `required` 校验提示，`"{label}"` 为表单域名称的占位符，会被自动替换                                                                     |
| type     | '{label}格式有误'     | 数据格式校验提示，`"{label}"` 为表单域名称的占位符，会被自动替换                                                                        |
| max      | '不得超过 {max} 个字' | `max` 校验提示，`"{max}"` 为最大字数的占位符，会被自动替换                                                                              |
| min      | '不得少于 {min} 个字' | `min` 校验提示，`"{min}"` 为最小字数的占位符，会被自动替换                                                                              |

设置示例：

```js
import { Form } from 'antx';

Form.setConfig({
  input: 'Please input ',
  required: 'Oops, {label} is needed',
});
```
